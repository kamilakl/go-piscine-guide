# Quest 11.4. List Last

## Название файла Go
`listlast.go`

---

## Условия задания 

| **Instruction** | **Перевод** |
|---------------------------|--------------|
| *Write a function `ListLast` that returns the `Data` interface of the last element of a linked list `l`.* | *Напишите функцию `ListLast`, которая возвращает поле `Data` последнего элемента списка `l`.* |

---

Функция должна вернуть данные (Data) из последнего узла списка.
Если список пуст — нужно вернуть nil.

### Пример

- **Ввод:** список содержит элементы `"three"`, `3`, `"1"`
- **Вывод:** `1`

---

## Решение

```go
package piscine

func ListLast(l *List) interface{} {
	if l.Head == nil {
		return nil
	}
	return l.Tail.Data
}
```

## Логика и пошаговый разбор кода

### 1. Объявление функции

```go
func ListLast(l *List) interface{} {
```
Эта строка объявляет функцию **ListLast**, которая принимает:

Функция принимает:
- `l *List` — указатель на структуру списка.
- Возвращает `interface{}` — данные в последнем узле.
Использование указателя (`*List`) важно, потому что ссылка даёт доступ к настоящему списку, а не его копии.


### 2. Проверка, пуст ли список

```go
if l.Head == nil {
	return nil
}
```

Объяснение:
- `l.Head == nil` означает, что в списке нет ни одного узла.
- В этом случае по условию задачи нужно вернуть `nil`.

#### Пример:

Пустой список:

```
Head → nil
Tail → nil
```

Функция вернёт `nil`.


### 3. Возврат данных последнего узла

```go
return l.Tail.Data
```

Так как:
- структура `List` хранит указатель на хвост,
- `Tail` всегда указывает на последний элемент,
получить последние данные можно за **O(1)**.

#### Пример:

`Head → ["three"] → [3] → ["1"] ← Tail`

`l.Tail.Data == "1"`

### Пошаговый пример работы

#### Пример 1: список НЕ пуст

`Head → ["three"] → [3] → ["1"] ← Tail`

1. Head != nil → список не пуст.
2. Возвращаем Tail.Data.
3. Вывод: 1

#### Пример 2: список пуст

```go
Head → nil
Tail → nil
```

1. Head == nil → список пуст.
2. Возвращаем nil.
3. Вывод: <nil>

## Заключение

- `ListLast` — простая функция, которая использует структуру `List`, уже содержащую указатель на `Tail`.
- Это делает операцию получения последнего элемента мгновенной **(O(1))**.
- Если список пуст — корректно возвращается `nil`.
- Функция демонстрирует удобство хранения `Tail` в односвязном списке: без него пришлось бы идти через весь список.
