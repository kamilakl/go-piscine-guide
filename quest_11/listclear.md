# Quest 11.5. List Clear

## Название файла Go
`listclear.go`

---

## Условия задания 

| **Instruction**                                                               | **Перевод**                                                                          |
| ----------------------------------------------------------------------------- | ------------------------------------------------------------------------------------ |
| *Write a function `ListClear` that deletes all nodes from a linked list `l`.* | *Напишите функцию `ListClear`, которая удаляет все узлы из односвязного списка `l`.* |
| *Tip: assign the list's pointer to nil.*                                      | *Подсказка: установите указатели списка в `nil`.*                                    |


---

Функция должна полностью очистить список, удалив все его элементы.
После вызова `ListClear(l)` список должен стать пустым:

```go
Head == nil  
Tail == nil
```

### Пример

- До очистки:

```go
I -> 1 -> something -> 2 -> <nil>
```

- После очистки:

```go
<nil>
```


---

## Решение

```go
package piscine

func ListClear(l *List) {
	l.Head = nil
	l.Tail = nil
}
```

## Логика и пошаговый разбор кода

### 1. Объявление функции
**В тройных скобках блок кода:**

```go
func ListClear(l *List) {
```
Эта строка объявляет функцию **ListClear**, которая принимает:

Функция принимает:
- `l *List` — указатель на список, который нужно очистить.
Использование указателя (`*List`) важно, потому что ссылка даёт доступ к настоящему списку, а не его копии.


### 2. Очистка головы списка

```go
l.Head = nil
```

- Убираем ссылку на первый элемент.
- После этого все остальные узлы становятся *недостижимыми* для программы.
- Но они ещё существуют в памяти — до тех пор, пока их не удалит **GC**.

### 3. Очистка хвоста

```go
l.Tail = nil
```

- Аналогично, убираем ссылку на последний элемент.
- Важно сбросить *оба* указателя, иначе список окажется в некорректном состоянии.

### 4. Почему этого достаточно?

В Go работает **сборщик мусора**.
Если на объект нет ссылок → он автоматически очищается из памяти.
После `Head = nil` и `Tail = nil`:

```go
[узлы списка]  --- no references --->  GC удалит позже
```

Очищать вручную по одному узлу (как в C) **не нужно**.

### Пошаговый пример работы

#### Был список:

```go
I -> 1 -> something -> 2 -> <nil>
Head → "I"
Tail → 2
```

#### После вызова:

```go
ListClear(link)
```

#### Станет:

```go
Head → nil  
Tail → nil
```

#### Вывод:

```go
<nil>
```

## Заключение

- `ListClear` — простая, но важная функция управления структурой данных.
- Она моментально очищает список, обнуляя `Head` и `Tail`.
- Сборщик мусора удалит все узлы автоматически.
- Такой подход безопасен и характерен для языков с **GC**.
- После очистки список корректно ведёт себя при новых операциях: можно снова добавлять элементы.
