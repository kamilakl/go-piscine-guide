# Quest 4.6 Sqrt (Целый квадратный корень)

## Название файла Go
`sqrt.go`

---

## Условия задания

| **Instruction** | **Перевод** |
|---------------------------|--------------|
| *Write a function that returns the square root of the `int` passed as parameter, if that square root is a whole number. Otherwise it returns `0`.* | *Напишите функцию, которая возвращает квадратный корень числа, переданного в качестве параметра, если этот корень является целым числом. В противном случае функция должна вернуть `0`.* |

---

Нужно написать функцию, которая находит квадратный корень числа, но возвращает значение **только если оно целое**.  
Если корень не является целым (например √10 = 3.16), то функция должна вернуть `0`.

---

### Пример

| Входное значение | Ожидаемый результат | Объяснение |
|------------------|---------------------|-------------|
| 4 | 2 | √4 = 2 → целое число |
| 9 | 3 | √9 = 3 → целое число |
| 10 | 0 | √10 ≈ 3.16 → не целое |
| 16 | 4 | √16 = 4 → целое число |
| 5 | 0 | √5 ≈ 2.23 → не целое |

---

## Решение

```go
package piscine

func Sqrt(nb int) int {
	if nb < 0 {
		return 0
	}
	for i := 0; i*i <= nb; i++ {
		if i*i == nb {
			return i
		}
	}
	return 0
}
```

---

## Ключевые понятия

| **Термин или инструмент** | **Объяснение** |
|----------------------------|----------------|
| **Квадратный корень (Square Root)** | Обратная операция к возведению в квадрат |
| **Целое число** | Число без десятичной части (…, -2, -1, 0, 1, 2, 3 …). В Go тип `int` используется для представления целых чисел. |
| **Цикл `for`** | Используется для итераций: в каждой итерации проверяем, не равно ли `i*i` исходному числу. |
| **Оператор `==` (равно)** | Проверяет, равны ли два значения. Если `true`, то функция возвращает `i`. |

---

### Как работает проверка квадратного корня

Вместо того чтобы использовать математическую функцию `sqrt()` из библиотеки (что запрещено в этой задаче),  
мы перебираем все возможные числа `i`, начиная с 0, и проверяем:

- Если `i*i` равно `nb`, значит `i` — это целый квадратный корень.
- Если `i*i` превышает `nb`, значит дальше проверять нет смысла — возвращаем 0.

---

## Логика и пошаговый разбор кода

### 1. Объявление функции

```go
func Sqrt(nb int) int {
```

Функция принимает одно целое число `nb` и возвращает его целый квадратный корень (тип `int`).

---

### 2. Проверка отрицательных чисел

```go
if nb < 0 {
	return 0
}
```

- Для отрицательных чисел квадратного корня в множестве **реальных чисел** не существует, поэтому сразу возвращаем `0`.

---

### 3. Перебор возможных корней

```go
for i := 0; i*i <= nb; i++ {
	if i*i == nb {
		return i
	}
}
```
- Начинаем с `i = 0` и проверяем каждое целое число.  
- Если квадрат `i` равен `nb`, то нашли точный корень.  
- Если `i*i` становится больше `nb`, цикл завершается.

---

### 4. Возврат 0, если корень не найден

```go
return 0
```

- Если не было найдено ни одного `i`, для которого `i*i == nb`, значит число не является полным квадратом (например, 8, 10, 12).  
  Возвращаем `0`.

---

## Пример выполнения

| Шаг | Значение `i` | Проверка `i*i` | Действие |
|------|---------------|----------------|-----------|
| 1 | 0 | 0 == 9 ❌ | продолжаем |
| 2 | 1 | 1 == 9 ❌ | продолжаем |
| 3 | 2 | 4 == 9 ❌ | продолжаем |
| 4 | 3 | 9 == 9 ✅ | возвращаем 3 |

Результат: **3**

---
## Заключение
Здесь презентовано наивное решение этой задачи: перебор чисел, пока не найдем то, квадрат которого равен `nb`. 
От наивного перебора можно прийти к более быстрым методам: например к методу Герона,который начинает с примерного корня и приближать результат итерационно.
Это тот же принцип поиска корня, только реализованный более эффективно.

Начинать с наивного решения полезно, потому что оно помогает разобраться в логике задачи без лишних усложнений. 
В более сложных задачах наивное решение может быть отправной точкой для оптимизированных решений.
